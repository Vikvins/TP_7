# TP_7 - Архитектура, слои и DDD-lite

## Цель работы
- Научиться строить слоистую архитектуру
- Реализовать доменные модели
- Написать use-case
- Разделить инфраструктуру
- Проверить поведение с помощью тестов

## Применённая архитектура
- Domain — Order, OrderLine, Money, OrderStatus и инварианты
- Application — PayOrderUseCase для выполнения оплаты
- Infrastructure — InMemoryOrderRepository, FakePaymentGateway
- Tests — проверка успешной оплаты, ошибок при пустом заказе, повторной оплате, изменении заказа после оплаты и корректного расчёта суммы

## Результат выполнения лабораторной:
- Чёткое разделение ответственности по слоям
- Возможность легко менять реализацию репозитория или платежной системы
- Доменные правила защищены от нарушений
- Возможность добавлять новые use-case без изменения доменной модели
